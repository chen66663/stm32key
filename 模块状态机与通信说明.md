# 模块状态机与通信说明

本文档汇总当前工程中各模块的线程配置、状态迁移矩阵、消息队列、Mail Queue 以及消息传递方式。

## 1. 线程配置

当前工程使用 CMSIS-RTOS v1 接口，线程定义在 `Core/main.c`，栈大小定义在 `Common/com_config.h`。

| 线程 | 入口函数 | 优先级 | 栈大小 |
| --- | --- | --- | --- |
| KEY | `app_key_taskEntry` | `osPriorityNormal` | `TASK_KEY_STACK_SIZE = 512U` |
| Power | `app_power_taskEntry` | `osPriorityNormal` | `TASK_POWER_STACK_SIZE = 512U` |
| CD | `app_cd_taskEntry` | `osPriorityNormal` | `TASK_CD_STACK_SIZE = 768U` |
| OLED | `app_oled_taskEntry` | `osPriorityNormal` | `TASK_OLED_STACK_SIZE = 768U` |

看门狗喂狗放在 RTX idle 线程中：

```c
void osRtxIdleThread(void *argument)
{
    (void)argument;

    for (;;)
    {
        bsp_iwdg_feed_dog();
    }
}
```

## 2. 队列与内存池

### 2.1 Message Queue

| 队列 | 类型 | 长度 | 用途 |
| --- | --- | --- | --- |
| `g_cdMsgQueue` | Message Queue | `QUEUE_CD_LEN = 8U` | KEY、Power、CD 定时器向 CD 模块发送消息 |
| `g_powerMsgQueue` | Message Queue | `QUEUE_POWER_LEN = 8U` | KEY 向 Power 模块发送消息 |
| `g_oledMsgQueue` | Message Queue | `QUEUE_OLED_LEN = 8U` | Power 向 OLED 模块发送上下电状态 |

普通消息使用 `AppMsg` 结构体。因为 CMSIS-RTOS v1 的 Message Queue 传递 32bit 数据，所以工程中先从 `g_appMsgPool` 申请一块 `AppMsg` 内存，再把该指针通过 `osMessagePut()` 放入队列。

发送流程：

1. `osPoolAlloc(g_appMsgPool)` 申请消息内存。
2. 拷贝 `AppMsg` 内容。
3. `osMessagePut(queue, (uint32_t)copy, 0U)` 发送消息指针。
4. 接收端 `osMessageGet()` 取出指针。
5. 处理完成后 `app_msg_free()` 释放回内存池。

### 2.2 Mail Queue

| 队列 | 类型 | 长度 | 用途 |
| --- | --- | --- | --- |
| `g_cdOledMailQueue` | Mail Queue | `MAIL_CD_OLED_LEN = 8U` | CD 向 OLED 发送显示数据 |

CD 到 OLED 按要求使用 Mail Queue，不通过普通 Message Queue。

CD 端：

```c
mail = (OledMail *)osMailAlloc(g_cdOledMailQueue, 0U);
osMailPut(g_cdOledMailQueue, mail);
```

OLED 端：

```c
evt = osMailGet(g_cdOledMailQueue, 0U);
osMailFree(g_cdOledMailQueue, mail);
```

## 3. 消息结构

### 3.1 AppMsg

```c
typedef struct
{
    ModuleId    srcModule;
    ModuleId    dstModule;
    MsgId       msgId;
    uint8_t     opt0;
    uint8_t     opt1;
    uint8_t     opt2;
    uint16_t    value;
} AppMsg;
```

| 字段 | 说明 |
| --- | --- |
| `srcModule` | 来源模块，例如 `MODULE_KEY`、`MODULE_POWER` |
| `dstModule` | 目标模块，例如 `MODULE_CD`、`MODULE_OLED` |
| `msgId` | 消息类型，例如 `MSG_ID_KEY_EVENT`、`MSG_ID_POWER_STATE` |
| `opt0` | 预留扩展字段，当前未使用 |
| `opt1` | 预留扩展字段，当前未使用 |
| `opt2` | 预留扩展字段，当前未使用 |
| `value` | 当前主要数据字段，用于传递按键事件、电源状态、CD 内部事件等 |

`opt0`、`opt1`、`opt2` 当前发送时均填 `0U`，暂未使用。后续可以扩展为按键持续时间、错误码、模式编号、来源通道等。

### 3.2 OledMail

```c
typedef struct
{
    ModuleId    srcModule;
    ModuleId    dstModule;
    MsgId       msgId;
    SysPower    powerState;
    SysState    cdState;
    uint8_t     cdDisplay;
    uint16_t    music;
} OledMail;
```

| 字段 | 说明 |
| --- | --- |
| `srcModule` | 来源模块，当前为 `MODULE_CD` |
| `dstModule` | 目标模块，当前为 `MODULE_OLED` |
| `msgId` | 当前为 `MSG_ID_CD_STATE` |
| `powerState` | 电源状态 |
| `cdState` | CD 状态 |
| `cdDisplay` | CD 显示模式，例如普通、快速上一曲、快速下一曲 |
| `music` | 当前歌曲号，范围 1 到 100 |

## 4. KEY 模块

KEY 模块没有状态机，只负责按键扫描、消抖、长短按识别和消息发送。

主要参数：

| 参数 | 值 | 说明 |
| --- | --- | --- |
| `KEY_SCAN_INTERVAL_MS` | `10U` | 每 10ms 扫描一次按键 |
| `KEY_PRESS_SAMPLES` | `3U` | 连续 3 次有效判定按下，即 30ms 消抖 |
| `KEY_LONG_PRESS_MS` | `1700U` | 1.7s 长按阈值 |
| `KEY_LONG_MARGIN_MS` | `80U` | 长短按边界保护时间 |

按键事件映射：

| 按键事件 | 目标模块 | 说明 |
| --- | --- | --- |
| `EV_KEY_WK_UP_SHORT` | Power | WK_UP 短按，上电 |
| `EV_KEY_WK_UP_LONG` | Power | WK_UP 长按，下电 |
| `EV_KEY_WK_UP_OFF` | Power | WK_UP 松手事件 |
| `EV_KEY_0_SHORT` | CD | KEY0 短按，Load/Eject |
| `EV_KEY_0_LONG` | CD | KEY0 长按，Previous |
| `EV_KEY_0_OFF` | CD | KEY0 松手，结束上一曲连续动作 |
| `EV_KEY_1_SHORT` | CD | KEY1 短按，Play/Pause |
| `EV_KEY_1_LONG` | CD | KEY1 长按，Next |
| `EV_KEY_1_OFF` | CD | KEY1 松手，结束下一曲连续动作 |

## 5. Power 模块状态迁移矩阵

Power 状态：

```c
SYS_POWER_OFF
SYS_POWER_ON
```

Power 事件：

```c
POWER_EVENT_KEY_ON
POWER_EVENT_KEY_OFF
POWER_EVENT_ERROR
POWER_EVENT_TOGGLE
```

状态迁移矩阵：

| 当前状态 | KEY_ON | KEY_OFF | ERROR | TOGGLE |
| --- | --- | --- | --- | --- |
| `SYS_POWER_OFF` | `SYS_POWER_ON` | `SYS_POWER_OFF` | `SYS_POWER_OFF` | `SYS_POWER_ON` |
| `SYS_POWER_ON` | `SYS_POWER_ON` | `SYS_POWER_OFF` | `SYS_POWER_OFF` | `SYS_POWER_OFF` |

实际按键映射：

| KEY 事件 | Power 事件 |
| --- | --- |
| `EV_KEY_WK_UP_SHORT` | `POWER_EVENT_KEY_ON` |
| `EV_KEY_WK_UP_LONG` | `POWER_EVENT_KEY_OFF` |

Power 状态变化后会通知 CD 和 OLED：

| 目标模块 | 队列 | 消息 |
| --- | --- | --- |
| CD | `g_cdMsgQueue` | `MSG_ID_POWER_STATE` |
| OLED | `g_oledMsgQueue` | `MSG_ID_POWER_STATE` |

Power 只做模拟上下电，不真实关闭硬件电源。

## 6. CD 模块状态迁移矩阵

CD 状态：

```c
SYS_STATE_POWER_OFF
SYS_STATE_NO_DISC
SYS_STATE_LOADING
SYS_STATE_EJECTING
SYS_STATE_STOP
SYS_STATE_PLAY
SYS_STATE_PAUSE
```

CD 事件：

```c
MATRIX_EVENT_POWER_ON
MATRIX_EVENT_POWER_OFF
MATRIX_EVENT_LOAD_EJECT
MATRIX_EVENT_DISC_ACTION_DONE
MATRIX_EVENT_PLAY_PAUSE
MATRIX_EVENT_PREVIOUS
MATRIX_EVENT_NEXT
MATRIX_EVENT_KEY_RELEASE
```

状态迁移矩阵：

| 当前状态 | Power On | Power Off | Load/Eject | Done | Play/Pause | Previous | Next | Release |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `SYS_STATE_POWER_OFF` | 恢复上次状态或 `STOP` | `POWER_OFF` | `POWER_OFF` | `POWER_OFF` | `POWER_OFF` | `POWER_OFF` | `POWER_OFF` | `POWER_OFF` |
| `SYS_STATE_NO_DISC` | `NO_DISC` | `POWER_OFF` | `LOADING` | `NO_DISC` | `NO_DISC` | `NO_DISC` | `NO_DISC` | `NO_DISC` |
| `SYS_STATE_LOADING` | `LOADING` | `POWER_OFF` | `LOADING` | `STOP` | `LOADING` | `LOADING` | `LOADING` | `LOADING` |
| `SYS_STATE_EJECTING` | `EJECTING` | `POWER_OFF` | `EJECTING` | `NO_DISC` | `EJECTING` | `EJECTING` | `EJECTING` | `EJECTING` |
| `SYS_STATE_STOP` | `STOP` | `POWER_OFF` | `EJECTING` | `STOP` | `PLAY` | `STOP` | `STOP` | `STOP` |
| `SYS_STATE_PLAY` | `PLAY` | `POWER_OFF` | `EJECTING` | `PLAY` | `PAUSE` | `PLAY` | `PLAY` | `PLAY` |
| `SYS_STATE_PAUSE` | `PAUSE` | `POWER_OFF` | `EJECTING` | `PAUSE` | `PLAY` | `PAUSE` | `PAUSE` | `PAUSE` |

CD 动作说明：

| 动作 | 说明 |
| --- | --- |
| Power On | 恢复上次状态；如果上次是异常中间态，则恢复为 `STOP` |
| Power Off | 保存当前有效状态到 `s_lastState` |
| Load | 进入 `LOADING`，启动 3s 软件定时器 |
| Load Done | 装碟完成，进入 `STOP`，歌曲号置为 1 |
| Eject | 进入 `EJECTING`，启动 3s 软件定时器 |
| Eject Done | 退碟完成，进入 `NO_DISC`，歌曲号置为 1 |
| Play/Pause | `STOP -> PLAY`，`PLAY -> PAUSE`，`PAUSE -> PLAY` |
| Previous | 歌曲号减 1，低于 1 回绕到 100 |
| Next | 歌曲号加 1，超过 100 回绕到 1 |
| Key Release | 松手后停止对应方向的连续拨曲 |

Previous/Next 行为：

| CD 状态 | Previous/Next 行为 |
| --- | --- |
| `STOP` | No Action |
| `NO_DISC` | No Action |
| `PLAY` | 单次上一曲或下一曲 |
| `PAUSE` | 每 0.5s 连续上一曲或下一曲，直到对应按键松开 |

CD 使用两个 RTOS 软件定时器：

| 定时器 | 类型 | 时间 | 用途 |
| --- | --- | --- | --- |
| `s_discTimer` | One-shot | `CD_DISC_ACTION_MS = 3000U` | Load/Eject 3s 模拟 |
| `s_repeatTimer` | Periodic | `CD_REPEAT_INTERVAL_MS = 500U` | Pause 状态下连续切歌 |

## 7. OLED 模块状态迁移矩阵

OLED 状态：

```c
OLED_STATE_OFF
OLED_STATE_READY
OLED_STATE_UPDATE
OLED_STATE_ERROR
```

OLED 事件：

```c
OLED_EVENT_POWER_OFF
OLED_EVENT_POWER_ON
OLED_EVENT_CD_UPDATE
OLED_EVENT_REFRESH
```

状态迁移矩阵：

| 当前状态 | Power Off | Power On | CD Update | Refresh |
| --- | --- | --- | --- | --- |
| `OLED_STATE_OFF` | `OFF` | `READY` | `OFF` | `OFF` |
| `OLED_STATE_READY` | `OFF` | `READY` | `UPDATE` | `UPDATE` |
| `OLED_STATE_UPDATE` | `OFF` | `READY` | `UPDATE` | `UPDATE` |
| `OLED_STATE_ERROR` | `ERROR` | `ERROR` | `ERROR` | `ERROR` |

OLED 接收两类数据：

| 来源 | 通信方式 | 内容 |
| --- | --- | --- |
| Power | `g_oledMsgQueue` | 上电、下电状态 |
| CD | `g_cdOledMailQueue` | CD 状态、显示模式、歌曲号 |

OLED 显示规则：

| 条件 | 显示 |
| --- | --- |
| Power Off | 清屏，不显示内容 |
| Power On | 第一行显示 `Power On` |
| CD 正常状态 | 第二行显示 `NO DISC`、`LOADING`、`EJECTING`、`STOP`、`PLAY`、`PAUSE` |
| CD 连续上一曲 | 第二行显示 `Fast Previousing` |
| CD 连续下一曲 | 第二行显示 `Fast Nexting` |
| 歌曲号 | 第三行显示 `Music ###` |

Power Off 时 OLED 会先清屏再刷新，因此屏幕保持空白。

## 8. 模块间消息流

### 8.1 开机

```text
KEY 检测 WK_UP 短按
    -> g_powerMsgQueue
    -> Power 状态切到 SYS_POWER_ON
    -> Power 发送 MSG_ID_POWER_STATE 给 CD
    -> Power 发送 MSG_ID_POWER_STATE 给 OLED
    -> CD 恢复上次状态
    -> OLED 点亮并显示状态
```

### 8.2 关机

```text
KEY 检测 WK_UP 长按 1.7s
    -> g_powerMsgQueue
    -> Power 状态切到 SYS_POWER_OFF
    -> Power 通知 CD
    -> Power 通知 OLED
    -> CD 保存当前状态并进入 POWER_OFF
    -> OLED 清屏
```

### 8.3 装碟/退碟

```text
KEY0 短按
    -> g_cdMsgQueue
    -> CD 收到 MATRIX_EVENT_LOAD_EJECT
    -> 进入 LOADING 或 EJECTING
    -> 启动 3s 软件定时器
    -> 定时器到期发送 MATRIX_EVENT_TIMER_DISC_DONE
    -> CD 状态变为 STOP 或 NO_DISC
    -> CD 通过 Mail Queue 通知 OLED
```

### 8.4 播放/暂停

```text
KEY1 短按
    -> g_cdMsgQueue
    -> CD 收到 MATRIX_EVENT_PLAY_PAUSE
    -> STOP 进入 PLAY
    -> PLAY 进入 PAUSE
    -> PAUSE 进入 PLAY
    -> CD 通过 Mail Queue 通知 OLED
```

### 8.5 上一曲/下一曲

```text
KEY0 长按
    -> CD 收到 MATRIX_EVENT_PREVIOUS

KEY1 长按
    -> CD 收到 MATRIX_EVENT_NEXT
```

在 `PLAY` 状态下，只切一首。

在 `PAUSE` 状态下，CD 启动 0.5s 周期软件定时器，持续切歌。对应按键松开后，CD 停止连续切歌。

## 9. 当前架构总结

当前工程满足以下设计点：

| 要求 | 当前实现 |
| --- | --- |
| CMSIS-RTOS v1 接口 | 使用 `cmsis_os.h`、`osThreadDef`、`osMessageQDef`、`osMailQDef`、`osTimerDef` |
| 分层结构 | Core、Common、BSP、App 分层 |
| KEY 无状态机 | KEY 只扫描和发消息 |
| Power 有状态机 | `s_powerStateMatrix` |
| CD 有状态机 | `g_stateMatrix` + `s_actionMatrix` |
| OLED 有状态机 | `s_oledStateMatrix` |
| 模块低耦合 | 模块通过队列传递消息 |
| CD 到 OLED 用 Mail Queue | `g_cdOledMailQueue` |
| Power 模拟上下电 | 不真实关闭电源，只广播状态 |
| Power Off 屏幕不显示 | OLED 清屏并刷新 |
| 看门狗放 idle 线程 | `osRtxIdleThread()` 喂狗 |
